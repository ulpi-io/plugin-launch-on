# Launch on Glassdoor

## Goal

Create `.ulpi/launch/glassdoor/` with `ROUTE-AND-ELIGIBILITY.md`, `BENCHMARK.md`, `EMPLOYER-PROFILE.md`,
`REVIEW-AND-INTERVIEW-OPERATIONS.md`, `JOBS-AND-CANDIDATE-EXPERIENCE.md`, `PRIVACY-AND-GOVERNANCE.md`,
`CHECKLIST.md`, `analytics.md`, and shared positioning.

## Workflow

### 1. Resolve employer authority and route

Ask for legal/display name, verified company facts, actual locations/team/work model, careers/jobs owner, employer
brand assets, HR/legal response owner, candidate-data policy, existing Glassdoor URL, review posture, and budget.
Browse the live employer route and return `READY TO CLAIM/UPDATE`, `READY FOR EMPLOYER PRESENCE`, or `BLOCKED`.

### 2. Benchmark five employers

Choose five peers by role market, size, region, industry, and work model. Record observed story, facts, jobs, review
themes, interview guidance, response approach, visual proof, candidate CTA, and gaps. Do not infer confidential staff,
compensation, culture, or ratings.

### 3. Prepare truthful employer materials

After authorized form capture, map every profile field to approved evidence: company/about, locations, work model,
mission, product/team facts, culture/values, benefits only if accurate, DEI/EEO claims only with approval, logo/media,
careers URLs, and actual jobs. Create a factual review/interview response framework that routes safety, legal, and
personnel issues without identifying reviewers.

### 4. Operate ethically

Present fields, assets, jobs, public responses, paid products, and named owners before action. Track profile status,
qualified applies, source quality, time-to-fill, candidate experience, review themes, response time, recurring issues,
and employer-brand spend. End `LIVE — GLASSDOOR EMPLOYER PRESENCE GOVERNED`.
