# Launch on Appvizer

## Goal

Create `.ulpi/launch/appvizer/` with `ROUTE.md`, `BENCHMARK.md`, `PROFILE-PACKAGE.md`, `FEATURE-AND-PRICING.md`,
`MEDIA-AND-LOCALIZATION.md`, `REVIEW-OPERATIONS.md`, `CHECKLIST.md`, `analytics.md`, and shared positioning.

## Workflow

### 1. Resolve category and route

Ask for product URL/name, buyer/problem/outcome, categories, features, integrations, pricing/trial, languages,
countries, screenshots/demo, support, approved proof, review candidates, existing Appvizer URL, and budget. Browse
the live category/product/vendor surface. Return `READY TO CREATE PROFILE`, `READY TO CLAIM/UPDATE`, or `BLOCKED`.

### 2. Benchmark five live peers

Choose peers by category, target buyer, maturity, price band, and region. Create an evidence table for category,
positioning, feature expression, visual sequence, pricing transparency, review language, comparison context, and CTA.
Mark each datum `LIVE`, `EDITORIAL`, `PAID`, `INFERRED`, or `UNKNOWN`.

### 3. Prepare every field and asset

After authorized form observation, create a field-ready package for official name/domain/vendor, concise and expanded
description, buyer/use cases, categories/features, integrations, deployment, languages, pricing/trial, logo,
screenshots with outcome captions, video, support, security/compliance only with proof, and truthful competitor context.
Create localized variants only from approved source copy and include an approval table for all translated claims.

### 4. Launch and operate

Before any state-changing action, show exact fields, locales, assets, URLs, pricing, review outreach, and paid choice
for approval. Track approval, category visibility, sessions, conversions, assisted comparison visits, review quality,
local-market performance, support questions, and paid spend separately. End `LIVE — APPVIZER PROFILE OPERATING`.
