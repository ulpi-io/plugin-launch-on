# PeerSpot Benchmark, Category, and Copy

## Live five-profile gate

Use `$browse` against the live target category, exactly five same-job products, official vendor sites, and at
least one relevant comparison page. Record timestamp/timezone, category URL, filters, sort/order, product count,
selection rationale, URLs, and any promoted/vendor-program treatment. On block use `$browse-stealth`; if it too
fails, keep the benchmark blocked.

PeerSpot's public homepage observed July 23, 2026 described reviews as verified recent-user feedback, typically
around 600 words, including pros and cons. Use that to assess content requirements, not to paraphrase a review as
vendor copy.

## What to compare

- category label and buyer intent
- overview/description and target deployment/customer
- review count/recency and reviewer company/role context
- review themes: use case, pros, cons, ROI, pain, alternatives, implementation and support
- product/technical features, integrations, security, pricing, proof, and media
- comparison links, alternatives, community questions, and vendor-site consistency

## Copy

Write original buyer-job, technical/workflow, and proof/reassurance angles. Every factual statement needs a
first-party source. Explain mechanisms and boundaries rather than superlatives. Do not name competitors in the
main description unless the live PeerSpot surface explicitly supports an accurate comparison relationship.

## Marketplace and LLM claims

PeerSpot's vendor form requests AWS and Google Cloud Marketplace status and offers marketplace/LLM areas of
interest. Mark status only when the canonical public listing exists. Treat any marketplace or LLM influence
program as commercial/plan-specific until live terms confirm it.
