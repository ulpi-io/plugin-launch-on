# Launch on Bing Places

## Goal

Create `.ulpi/launch/bing-places/` with `ELIGIBILITY.md`, `ROUTE.md`, `BENCHMARK.md`, `LISTING-PACKAGE.md`,
`VERIFICATION.md`, `REVIEW-OPERATIONS.md`, `GOVERNANCE.md`, `CHECKLIST.md`, `analytics.md`, and shared positioning.

## Workflow

### 1. Establish eligibility and authority

Ask for official name, real local/service-area operation, address visibility decision, service area, category, hours,
phone, website, services, actual photos, owner/managers, verification contact, existing Bing/other map profiles, and
approved proof. Return `READY TO CLAIM/VERIFY`, `READY TO CREATE`, or `NOT ELIGIBLE — NO LOCAL OPERATION`.

### 2. Benchmark five live peers

Select five comparable local profiles by category, geography, and customer model. Record observed information,
completeness, visuals, reviews, offers/services, and CTA, then identify truthful gaps without copying content.

### 3. Build the field-ready listing

After authorized sign-in, capture all live fields/validation and prepare exact values for business name, category,
address/service area, pin, hours/special hours, phone, website, services, attributes, description, photos/logo, and
verification. Maintain a source ledger plus address-privacy and holiday-hours plan.

### 4. Verify and operate

Present all fields, verification contact, address display, assets, and review-response owner for approval. Track
verification, Bing search/map views, calls, directions, website clicks, local leads, reviews, response time, and
accuracy changes. End `LIVE — BING PLACES PROFILE GOVERNED`.
