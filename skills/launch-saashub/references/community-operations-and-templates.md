# SaaSHub operations and templates

## Ethical feedback and comments

Use one neutral, post-use eligibility rule across the full eligible cohort. Invite customers to share an honest
experience, no requested score or sentiment; honor suppressions and privacy obligations. Never write a user review
or comment for someone else, promise an outcome, or make participation conditional on support, plan, spend, or
relationship.

## Submission checklist

```md
# SaaSHub submission checklist

- [ ] Released, English, own-domain product passes every current eligibility gate
- [ ] Duplicate and existing listing search complete
- [ ] Exactly five live peer profiles, official sites, and relevant comparison evidence recorded
- [ ] Current form fields and limits captured in exact order
- [ ] Categories and listed competitors match the primary buyer job
- [ ] Pricing, platforms, release date, features, media, and claims are source-backed
- [ ] Verification route and authorized vendor owner confirmed
- [ ] No review/comment/comparison manipulation; no private data in artifacts
- [ ] Human approval received before any external action
```

## Operating metrics

| Metric | Evidence | Cadence | Guardrail |
|---|---|---|---|
| Listing/category referral | UTM/referrer + first-party event | weekly | privacy-safe aggregation |
| CTA conversion | trial/demo/docs event | weekly | separate test traffic |
| Comparison referral | page/referrer data | monthly | no false competitor associations |
| Verification/freshness | owner ledger + quarterly prompt | quarterly/release | recheck every public claim |
| Feedback/comments | invite and response log | monthly | neutral solicitation only |
| Featured placement | written terms + invoice | campaign | never imply organic rank |
```
