# Launch on Serchen

## Goal

Create `.ulpi/launch/serchen/` with `ROUTE.md`, `BENCHMARK.md`, `PROFILE-PACKAGE.md`,
`PRICING-AND-FEATURES.md`, `MEDIA-AND-COMPARISONS.md`, `REVIEW-OPERATIONS.md`, `CHECKLIST.md`, `analytics.md`, and
shared positioning.

## Workflow

### 1. Resolve the listing route and category

Ask for name/domain, buyer/problem/outcome, category, features, integrations, pricing/trial, regions/languages,
media/demo, support, approved proof, review candidates, existing profile, and budget. Browse Get Listed and relevant
category pages. Return `READY TO CREATE`, `READY TO CLAIM/UPDATE`, or `BLOCKED — EVIDENCE/ACCOUNT NEEDED`.

### 2. Benchmark five live peers

Choose five closest by product type, buyer, category, price band, and deployment. Create an evidence table with public
profile pattern, buyer language, visual/content sequence, ratings/review themes, comparison placement, guide context,
and CTA. Label observations `LIVE`, `EDITORIAL`, `PAID`, `INFERRED`, or `UNKNOWN`.

### 3. Prepare field-ready content

After authorized form inspection, map every live field: official vendor/product, concise description, expanded
overview, category/tags, use cases, features, integrations, deployment, pricing/trial, regions/languages, logo,
screenshots/captions, video, support, official links, and approved comparison context. Build a source ledger and keep
pricing and screenshots synchronized with the product site.

### 4. Publish and operate

Present exact fields, assets, URLs, review outreach, and any paid/editorial option for approval. Track profile status,
category/comparison visits, sessions, conversions, review quality, buyer-guide referrals, support issues, updates, and
paid spend separately. End `LIVE — SERCHEN PROFILE OPERATING`.
