# Launch on Launching Next

## Goal

Create `.ulpi/launch/launching-next/` with `ROUTE-AND-FIT.md`, `BENCHMARK.md`, `COPY-PACKET.md`,
`TAG-TAXONOMY.md`, `PRIVATE-CONTEXT.md`, `FORM-CAPTURE.md`, `COMMERCIAL-DECISION.md`, `analytics.md`, and positioning.

## Workflow

### 1. Confirm submission fit

Return `FREE SUBMISSION READY`, `EXPEDITED OPTION TO REVIEW`, `NEEDS CLEARER POSITIONING`, or `NOT A FIT`. Verify the
startup/site is live and functional, the authorized representative understands the private marketing-spend question,
and the product can be described within the headline/tag constraints.

### 2. Benchmark exactly five peers

Use `$browse` to examine five live comparable listings. Record their headline word count, full-description structure,
tag choice, page CTA, and reader-facing product category. Create original 5–8-word headline candidates and an evidence-
based 2,500-character-max description; do not mimic wording.

### 3. Build complete form-ready copy

Prepare canonical Startup Name/URL; headline candidates with verified word count; full description with verified
character count; 5–10 distinct tags; truthful type selection; approved private marketing-spend range; private submitter
name/email; newsletter preference; public support URL; source register; and correction owner. Keep private inputs out of
public launch copy.

### 4. Verify the optional expedited path

If the user considers the paid path, re-capture exact price, currency, tax, payment/refund, SLA wording, scope, and
confirmation fields. Record why speed is needed and explicitly note that it is not a publication guarantee.

### 5. Submit with approval and maintain

Present the exact live form, every count, field-to-source map, private-data boundary, newsletter choice, and optional
payment terms for approval. After authorized submission, save confirmation, track publication/correction status, verify
the final public page/link, and measure qualified referrals and conversion. End `LIVE — LAUNCHING NEXT RUNBOOK ACTIVE`.
