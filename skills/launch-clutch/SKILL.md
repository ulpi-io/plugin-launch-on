# Launch on Clutch

## Goal

Create `.ulpi/launch/clutch/` with `ELIGIBILITY.md`, `BENCHMARK.md`, `PROFILE-PACKAGE.md`, `PORTFOLIO-PACKAGE.md`,
`REVIEW-OPERATIONS.md`, `VERIFICATION-AND-PAID.md`, `CHECKLIST.md`, `analytics.md`, and shared positioning.

## Workflow

### 1. Prove fit and resolve the route

Ask for services, delivery model, target buyers, minimum project size, locations, industries, portfolio permission,
client-review candidates/consent, legal entity, verification owner, budget, and existing Clutch URL. Browse the
applicable provider route. Return `READY TO CREATE PROVIDER PROFILE`, `READY TO CLAIM/UPDATE`, or `NOT ELIGIBLE`.

### 2. Benchmark exactly five providers

Choose five peers by service line, buyer, geography, delivery model, and price band. Record observed categories,
service claims, review themes, case-study structure, project outcomes, proof, buyer CTA, visibility labels, and gaps.
Do not copy competitor language or infer private commercial data.

### 3. Build field-ready, evidence-backed content

After authorized form observation, create a map for every current profile field: official entity/domain, service lines,
positioning, about, locations, team, project economics only if approved, industries, technology, logo/media, portfolio
case studies, client permission/evidence, contacts, and support. For each case study, distinguish verified public fact,
client-approved detail, and confidential material. Prepare review requests that preserve reviewer independence.

### 4. Operate with approval

Present exact fields, assets, client list/consent state, review outreach, verification choice, paid choice, cost, and
timing before action. Track profile status, verified reviews, category exposure, project-match leads, qualified
opportunities, win rate, revenue attribution, review health, and paid spend separately. End `LIVE — CLUTCH PROVIDER
OPERATIONS ACTIVE`.
